## What This Helper Is

`navigation.reveal()` is the high-level bounds helper for container-style navigation transitions.

It uses the same boundary link system as `bounds()` and `navigation.zoom()`, but instead of animating a floating matched element above the screen, it scales and masks the destination content from the source boundary into the destination container.

Use it when the destination screen should feel like it is revealed from the tapped source, especially for card, photo, or media-detail flows.

## Recommended Setup

Reveal requires three pieces: a source boundary, a matching destination boundary, and a navigation interpolator that calls `navigation.reveal()`.

### Source Setup

Mark the source with `Transition.Boundary.Trigger` and pass the reveal id through your navigation params.

```tsx
import Transition from "react-native-screen-transitions";

<Transition.Boundary.Trigger
  id={item.id}
  onPress={() => {
    navigation.navigate("Detail", { revealId: item.id });
  }}
>
  <Image source={item.image} style={styles.thumbnail} />
</Transition.Boundary.Trigger>
```

### Destination Setup

The destination screen must render a matching boundary for the same id. That destination boundary is the real container target for the reveal.

```tsx
function DetailScreen() {
  return (
    <Transition.Boundary.View id={item.id}>
      <Image source={item.image} style={styles.hero} />
    </Transition.Boundary.View>
  );
}
```

If the target element is nested inside a larger pressable or layout owner, use `Transition.Boundary.Target` inside the boundary owner to narrow the measured region.

### Navigation Setup

Call `navigation.reveal()` from the destination screen's `screenStyleInterpolator` and read the id from the active route params.

```tsx
<Stack.Screen
  name="Detail"
  component={DetailScreen}
  options={{
    navigationMaskEnabled: true,
    gestureEnabled: true,
    gestureDirection: ["vertical", "horizontal"],
    gestureProgressMode: "freeform",
    screenStyleInterpolator: ({ active, bounds }) => {
      "worklet";

      const id = active.route.params?.revealId;
      if (!id) return null;

      return bounds({ id }).navigation.reveal({
        borderRadius: 58,
        borderContinuous: true,
        maxSensitivity: 0.8,
        velocityDepth: 0.5,
      });
    },
  }}
/>
```

<Note>
  Reveal is intentionally opinionated: `navigation.reveal()` only works with the navigation mask enabled, and that mask requires `@react-native-masked-view/masked-view`. The helper accounts for Android and iOS mask performance internally, but if you need the lightest possible path, use `navigation.zoom()` instead; zoom does not require masked view.
</Note>

Install `@react-native-masked-view/masked-view`; without it, navigation mask styles cannot render.

## Gesture Behavior

Reveal reads the active gesture state from the transition runtime:

- pan directions can drag the revealed container away from the target
- `active.gesture.velocity` affects the dismiss handoff scale
- `gestureProgressMode: "freeform"` is usually the right choice when you want the reveal container to respond to drag without letting drag progress directly own the whole screen transition

```tsx
options={{
  gestureEnabled: true,
  gestureDirection: ["vertical", "horizontal"],
  gestureProgressMode: "freeform",
}}
```

Use `active.gesture.raw` if you derive dynamic runtime options such as `gestureSensitivity` from gesture movement.

## Options

Pass options directly to `navigation.reveal()` when you need to tune the opinionated defaults:

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `borderRadius` | `number` | Platform-specific reveal radius | Destination mask border radius. Reveal interpolates from the measured source radius to this value. |
| `borderContinuous` | `boolean` | `true` | Uses React Native's continuous border curve for the reveal mask. |
| `maxSensitivity` | `number` | `0.8` | Highest gesture sensitivity reveal applies before it reduces sensitivity during deeper drags. |
| `velocityDepth` | `number` | `0.5` | Velocity-driven depth for the dismiss scale orbit. Set `0` to remove the velocity depth effect. |
| `gestureProgressMode` | `"progress-driven" \| "freeform"` | `"freeform"` | Controls whether gesture displacement drives transition progress. Reveal defaults to freeform so drag can move the masked container without owning the full screen progress. |
| `disablePointerEventsTillElementTransition` | `boolean` | `true` | Blocks pointer-event pass-through on the inactive content until the source element handoff reaches progress `0.25`. This is intentional and most users cannot react fast enough to swipe during that brief handoff. Set `false` to disable the guard and keep the default behavior that allows pointer events to pass through. |
| `maskSizingMode` | `"auto" \| "transform" \| "size"` | `"auto"` | Controls how the navigation mask element resizes. Android defaults to transform-based resizing for performance; use `"size"` if transformed border radii look wrong and radius quality matters more. |

## Platform Notes

On Android, reveal uses a transform-based mask path by default to avoid masked-size animation issues. That keeps the API the same, but transformed masks can make large border radii look less natural. Use `maskSizingMode: "size"` to force width and height animation when border radius quality matters more than the Android optimization.

## Reveal vs Zoom

Use [Zoom](/navigation-zoom) when the matched element should float above the transition and the background content should scale behind it.

Use reveal when the destination content itself should grow out of the source and remain clipped by a navigation mask during the handoff.
